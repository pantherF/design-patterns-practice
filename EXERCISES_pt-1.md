A set of coding challenges covering everything up to (but not including) the SOLID principles. These are in C# since that's my strongest OOP language, with notes where a concept maps especially well to a language feature.

> The challenges were generated by Claude Sonnet using Brave AI.

---

## 🧩 OOP Basics

### Exercise 1 — Classes & Objects
**Topic:** Fields, methods, instances

```text
Create a `BankAccount` class with:
- Fields: owner (string), balance (decimal), accountNumber (string)
- A constructor that sets all three
- A Deposit(amount) method
- A Withdraw(amount) method that prevents overdraft (throw an exception if insufficient funds)
- A GetSummary() method that returns a formatted string

Then instantiate two different accounts and perform several operations on each.
```

**Stretch:** Add a static field `TotalAccounts` that increments each time a new account is created.

---

### Exercise 2 — Class Hierarchies & Inheritance
**Topic:** Superclass, subclass, method overriding

```text
Create an Animal base class with:
- Fields: Name, Age
- A virtual method Speak() that returns "..."

Then create at least three subclasses (Dog, Cat, Parrot) that:
- Override Speak() with their own sound
- Add at least one unique field or method each (e.g. Dog has a Breed field)

Finally, create a List<Animal> containing a mix of all three,
loop over it, and call Speak() on each.
```

## 🏛️ Pillars of OOP

### Exercise 3 — Abstraction
**Topic:** Modeling only what's relevant to context

```text
You're building a hospital system. Model a Patient class
that captures only the data relevant to a medical context
(ignore billing, social media, etc.).

Then model the same person as a Customer for a pharmacy
system — different fields, same real-world person.

Write both classes and a short Main() that creates one of each
and prints a summary for each context.
```

---

### Exercise 4 — Encapsulation
**Topic:** Private state, public interface

```text
Create a Temperature class that:
- Stores temperature internally in Celsius (private field)
- Exposes a Celsius property (get/set) with validation (no lower than -273.15)
- Exposes a Fahrenheit property (get only, computed from Celsius)
- Exposes a Kelvin property (get only, computed from Celsius)

No field should be directly accessible from outside the class.
```

**Stretch:** Add a static `FromFahrenheit(double f)` factory method.

---

### Exercise 5 — Interfaces & Abstraction (combined)
**Topic:** Contracts, `interface` keyword

```text
Define an interface IShape with:
- double Area()
- double Perimeter()
- string Describe()

Implement it in three classes: Circle, Rectangle, Triangle.

Write a method PrintShapeInfo(IShape shape) that works
for any shape without knowing its concrete type.

Then create a List<IShape> with mixed shapes and print
info for all of them.
```

---

### Exercise 6 — Inheritance (deeper)
**Topic:** Abstract classes, forced overrides

```text
Create an abstract class Vehicle with:
- Abstract method: string FuelType()
- Abstract method: int MaxSpeed()
- Concrete method: string Describe() — uses FuelType() and MaxSpeed()

Create subclasses: ElectricCar, PetrolMotorcycle, Bicycle.

Demonstrate that you cannot instantiate Vehicle directly,
but can use it as a reference type for the subclasses.
```

---

### Exercise 7 — Polymorphism
**Topic:** Runtime type detection, virtual dispatch

```text
Create a PaymentProcessor base class with a virtual
method Process(decimal amount) that logs "Processing payment..."

Create subclasses: CreditCardProcessor, PayPalProcessor, CryptoProcessor
— each overrides Process() with unique behavior/output.

Write a method RunPayments(List<PaymentProcessor> processors, decimal amount)
that calls Process() on each without any type-checking (no `is` or `as`).

Verify that the correct subclass method runs each time.
```

## 🔗 Relations Between Objects

### Exercise 8 — Dependency
**Topic:** Weakest relation — one class uses another temporarily

```text
Create a ReportGenerator class with a method
Generate(DataFormatter formatter) — it uses the formatter
only within that method call.

Create two DataFormatter subclasses: CsvFormatter and JsonFormatter,
each with a Format(string data) method.

Show that changing the formatter passed in changes the output,
without changing ReportGenerator at all.
```

---

### Exercise 9 — Association
**Topic:** Permanent link via field

```text
Create a Driver class and a Car class.
A Driver has a Car field (they're associated — the driver
knows about their car long-term).

Give Car a method Drive() and a property Owner.
Give Driver a method StartJourney() that uses their Car.

Show that the Driver and Car can exist independently —
you can reassign a Driver's car to a different Car object.
```

---

### Exercise 10 — Aggregation
**Topic:** "Has-a" where parts can exist independently

```text
Create a Department class and a Professor class.
A Department contains a List<Professor>.

Professors should be created independently and then
added to a department. Show that:
1. A professor can exist without a department
2. A professor can be in multiple departments
3. Removing a professor from a department doesn't destroy them
```

---

### Exercise 11 — Composition
**Topic:** "Has-a" where parts cannot exist without the whole

```text
Create a House class that is composed of Room objects.
Rooms are created inside the House constructor and
cannot be created or exist outside of it.

Give House a method AddRoom(string name, double sqMeters)
and a method GetTotalArea().

When the House object is destroyed (or goes out of scope),
the rooms should go with it — enforce this by making Room
a private nested class inside House.
```

## 🎯 Bonus Mini-Challenge — Putting It All Together

```text
Design a small Library system using all the relation types:

- Book (plain class with fields)
- ILendable interface (with Lend() and Return() methods)
- LibraryItem : abstract class implementing ILendable partially
- PhysicalBook and EBook extend LibraryItem
- Member class (associated with borrowed books via a List<LibraryItem>)
- Library class (aggregates Members, composes its Catalog of items)

Requirements:
- A Member can borrow and return items (polymorphically)
- The Library catalog is composed — items belong to the library
- Members exist independently of the library
- Print a summary of all borrowed items per member
```
